## 二、快速开始

### 1. 获取代码

```bash
git clone https://github.com/zhangjun123400/company-agent.git
cd company-agent
```

### 2. 配置凭据


健康检查：`http://localhost:3456/health`

## 三、配置说明

所有配置集中在 `config.env` 文件中，格式为 `字段名=值`。

### 必填项（必须配置，否则服务无法启动）

| 字段名 | 含义 | 示例值 |
|--------|------|--------|
| `FEISHU_PROJECT_PLUGIN_ID` | 飞书项目插件 ID | `MII_xxxxxxxxxxxx` |
| `FEISHU_PROJECT_PLUGIN_SECRET` | 飞书项目插件密钥 | `0154xxxxxxxxxxxx` |
| `FEISHU_PROJECT_USER_KEY` | 你的飞书项目用户标识 | `7649567855117192178` |
| `FEISHU_APP_ID` | 飞书机器人 App ID | `cli_xxxxxxxxxxxxx` |
| `FEISHU_APP_SECRET` | 飞书机器人 App Secret | `lvnncc4xxxxxxxxxxxxx` |
| `DEEPSEEK_API_KEY` | DeepSeek 大模型 Key | `sk-d7xxxxxxxxxxxxxxxxxxxx` |

### 可选项（有默认值，可不改）

| 字段名 | 含义 | 默认值 |
|--------|------|--------|
| `PROJECT_SPACE_KEY` | 飞书项目空间标识 | `aniwonder` |
| `AI_MODEL` | 大模型类型 | `deepseek` |
| `DEEPSEEK_BASE_URL` | DeepSeek API 地址 | `https://api.deepseek.com` |
| `PRD_REVIEW_TIMEOUT_DAYS` | PRD 超时提醒天数 | `3` |
| `CHECK_CRON` | 定时检查表达式 | `0 9 * * *` |

## 四、各平台凭据获取方法

### 4.1 飞书项目插件凭据

```
1. 打开飞书项目网页版 → https://project.feishu.cn
2. 左下角头像 → 开放平台
3. 创建插件（或使用已有插件）
4. 记录 Plugin ID 和 Plugin Secret
5. 填入 config.env 的 FEISHU_PROJECT_PLUGIN_ID 和 FEISHU_PROJECT_PLUGIN_SECRET
```

### 4.2 飞书项目 User Key

```
1. 打开飞书项目网页版
2. 左下角点击你的头像
3. 在弹窗中双击头像区域
4. User Key 已自动复制到剪贴板
5. 填入 config.env 的 FEISHU_PROJECT_USER_KEY
```

### 4.3 飞书机器人应用凭据

```
1. 打开飞书开放平台 → https://open.feishu.cn
2. 创建自建应用（或使用已有应用"智小协"）
3. 在应用首页获取 App ID 和 App Secret
4. 填入 config.env 的 FEISHU_APP_ID 和 FEISHU_APP_SECRET

此应用需要开通以下权限：
  - im:message（发送消息）
  - docx:document（创建文档）
  - wiki:wiki（读取知识库）
  - contact:contact:readonly（查询用户）
  - drive:drive.metadata:readonly（文档授权）
```

### 4.4 DeepSeek API Key

```
1. 打开 DeepSeek 平台 → https://platform.deepseek.com
2. 注册/登录 → API Keys → 创建新 Key
3. 复制 Key（sk- 开头）
4. 填入 config.env 的 DEEPSEEK_API_KEY
```

## 六、新增智能体

智能体（Agent）是执行具体任务的 AI 助手，挂载在飞书项目的流程节点上。当工作项流转到该节点时，对应的智能体会自动执行分析、评审等任务。

### 6.1 概述

每个智能体由以下信息定义：

| 字段 | 说明 | 示例 |
|------|------|------|
| **名称** | 智能体名字 | 硬件方案评审 |
| **节点** | 挂载的飞书项目流程节点 | 硬件方案设计 |
| **功能** | 智能体做什么（自然语言描述） | 读取硬件方案文档，从EMC、散热、结构三个方面评审 |
| **部门** | 归属部门 | 硬件部 |
| **prompt** | 系统提示词（定义智能体的身份和行为规则） | 自动生成 |
| **skills** | 技能列表（每项是自然语言描述的能力） | 自动生成 |

> 只需提供前 4 个字段（名称、节点、功能、部门），prompt 和 skills 由系统自动生成。后续可手动编辑 JSON 文件补充。

### 6.2 方式一：飞书对话新增（推荐，仅管理员）

给智小协发一条消息即可创建：

```
新增智能体：名称=硬件方案评审，节点=硬件方案设计，功能=读取硬件方案文档，从EMC、散热、结构三个方面进行评审，输出评审意见到飞书文档，部门=硬件部
```

系统自动执行以下步骤：

```
收到消息 → 校验管理员权限 → DeepSeek解析 → 生成完整配置 → 写入文件 → 热加载 → 回复"已创建"
```

**全程无需重启服务，新增后即时生效。**

> ⚠️ 仅管理员可新增智能体。普通用户发送此消息会收到"仅管理员可新增智能体"的回复。

### 6.3 方式二：手动添加配置文件

在 `agents/` 目录下新建一个 JSON 文件，文件名就是智能体名称：

```json
{
  "id": "硬件方案评审",
  "name": "硬件方案评审",
  "department": "硬件部",
  "node": "硬件方案设计",
  "description": "读取硬件方案文档，从EMC、散热、结构三方面评审",
  "prompt": "你是硬件方案评审专家，负责从电磁兼容、散热设计、结构合理性三个维度评审硬件方案...",
  "skills": ["EMC分析", "散热评审", "结构评审", "成本评估"],
  "tools": ["wiki:read", "docx:create", "im:send"],
  "output": "feishu_doc",
  "output_target": ["节点负责人"],
  "enabled": true,
  "timeout_ms": 300000
}
```

然后在 `agents/_registry.json` 中添加文件名：

```json
{
  "agents": [
    "需求分析.json",
    "技术可行性初评.json",
    "硬件方案评审.json"
  ]
}
```

保存后系统自动检测文件变化，热加载生效，无需重启。

### 6.4 配置后操作

#### 查看所有智能体

浏览器打开 `http://localhost:3456/api/agents`，返回所有已注册的智能体及其挂载节点。

#### 验证智能体是否生效

1. 在飞书项目中将一个工作项流转到智能体挂载的节点
2. 查看服务日志，确认智能体被触发
3. 检查飞书文档是否生成了报告

#### 修改智能体配置

直接编辑 `agents/` 目录下对应的 JSON 文件，保存后系统自动热加载（约 500ms 延迟）。

修改内容包括：
- 添加/删除 `skills`（技能）
- 修改 `prompt`（优化智能体的行为和输出）
- 更换挂载的 `node`（流程节点）
- 调整 `timeout_ms`（超时时间）

#### 禁用/启用智能体

将配置中的 `"enabled"` 改为 `false` 即可禁用，不影响其他智能体：

```json
"enabled": false
```

#### 删除智能体

管理员给智小协发消息：

```
删除智能体：硬件方案评审
```

或直接删除 `agents/` 下对应的 JSON 文件，并从 `_registry.json` 中移除该条目。

### 6.5 权限说明

| 操作 | 管理员 | 普通用户 |
|------|:---:|:---:|
| 新增智能体 | ✅ | ❌ |
| 删除智能体 | ✅ | ❌ |
| 修改智能体配置 | ✅ | ❌ |
| 查看智能体列表 | ✅ | ✅ |
| 触发分析（飞书项目节点） | ✅ | ✅ |
| 日常聊天（智小协） | ✅ | ✅ |

**管理员管理：**
- 初始管理员：部署时在 `config/admins.json` 中配置
- 新增管理员：管理员给智小协发 `"加xxx为管理员"`
- 移除管理员：管理员给智小协发 `"移除管理员xxx"`

### 6.6 架构说明

```
飞书项目节点流转
    → Webhook/轮询
      → 主智能体查询 "该节点挂载了哪些Agent"
        → 逐个执行（隔离）
          ├─ Agent A 执行成功 → 创建飞书文档 → 通知负责人
          ├─ Agent B 执行失败 → 记录日志 → 不影响Agent A
          └─ Agent C 超时     → 终止       → 不影响Agent A和B
```

每个智能体独立执行，互不影响。一个出错不会导致全部不可用。

## 七、部署到服务器

### 6.1 Linux / macOS 服务器

```bash
# 1. 安装 Node.js v18+
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
```

## 八、常见问题

**Q: 启动报错 "缺少必需配置"？**
A: 检查 `config.env` 文件是否存在，必填项是否都已填写。

**Q: 健康检查显示连接失败？**
A: 检查网络是否正常，飞书项目插件是否已安装到目标空间。

**Q: Wiki 读取失败？**
A: 检查 OAuth 授权是否已完成。访问 `http://localhost:3456/auth/feishu-login` 重新授权。

**Q: DeepSeek 分析失败？**
A: 检查 `DEEPSEEK_API_KEY` 是否正确，Key 余额是否充足。

**Q: 端口被占用？**
A: 修改启动命令 `PORT=3000 npm run dev` 使用其他端口。

**Q: 如何更新代码？**
A: `git pull && npm install && npm run dev`（重启服务）
