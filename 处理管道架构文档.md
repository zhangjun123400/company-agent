# 智慧智能体 · 处理管道 (Pipeline)

## 一、总览

```
┌──────────────────────────────────────────────────────────────────────┐
│                         智慧智能体 · Pipeline                         │
│                                                                      │
│  输入层                    处理层                    输出层            │
│  ────────                ────────                ────────             │
│                                                                      │
│  飞书项目 ──→  需求节点流转       ──→  主智能体调度  ──→  飞书文档      │
│  飞书 IM ──→  聊天/管理命令      ──→  Agent 执行    ──→  飞书消息      │

│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

## 二、三条核心 Pipeline

### Pipeline 1：自动分析（飞书项目触发）

```
① 飞书项目
   需求工作项流转到「需求评审」节点
   │
   ▼
② GET /cron/daily-check（每日轮询）
   或飞书项目自动化规则 → Webhook → POST /webhook/project
   │
   ▼
③ getWorkItemDetail()
   飞书项目 API: POST /open_api/{project_key}/work_item/{type}/query
   获取：需求名称、PRD链接、节点信息、负责人
   │
   ▼
④ extractPrdContent()
   从 fields 中提取 PRD Wiki 链接
   │
   ▼
⑤ readWikiByUrl()
   ├─ wiki_token.json 有效？ → 飞书 Wiki API 读取
   │   └─ Wiki API: GET /open-apis/wiki/v2/spaces/get_node?token=
   │       → 拿到 docId
   │         → Docx API: GET /open-apis/docx/v1/documents/{docId}/blocks
   │           → 提取纯文本 PRD 内容
   │
   └─ wiki_token.json 过期？
       → 自动发授权卡片给需求提出人（飞书 IM）
       → 返回 needAuth=true，等待授权后重试
   │
   ▼
⑥ Agent 并行执行（sandbox.ts 隔离）
   ├─ 需求分析 Agent (requirement-analyzer)
   │   └─ aiComplete() → DeepSeek API
   │       → 生成需求澄清问题清单
   │
   └─ 技术可行性初评 Agent (tech-feasibility-reviewer)
       └─ aiComplete() → DeepSeek API
           → 生成技术可行性初评报告
   │
   ▼
⑦ createFeishuDoc() ×2
   飞书文档 API: POST /open-apis/docx/v1/documents
   → 创建带格式的飞书文档
   │
   ▼
⑧ sendReportCard() + grantDocAccess()
   飞书 IM API: POST /open-apis/im/v1/messages
   → 发卡片消息通知负责人
   → 文档授权给接收人
```

### Pipeline 2：飞书 IM 聊天

```
① 同事在飞书给智小协发消息
   │
   ▼
② WebSocket 长连接接收
   @larksuiteoapi/node-sdk → im.message.receive_v1 事件
   │
   ▼
③ 消息分类（ws-client.ts）
   │
   ├─ 管理命令？
   │   ├─ "新增智能体" → createAgentFromMessage()
   │   │   ├─ isAdmin() 权限校验
   │   │   ├─ DeepSeek 解析自然语言 → AgentConfig
   │   │   ├─ registry.register() → 写入 agents/ 目录
   │   │   ├─ dispatcher.rebuild() → 热加载
   │   │   └─ 回复 "✅ 已创建"
   │   │
   │   ├─ "删除智能体" → registry.unregister() → 回复
   │   ├─ "查看智能体列表" → registry.getAll() → 回复
   │   └─ "授权 [文档名]" → sendAuthCard() → 回复授权卡片
   │
   ├─ 分析命令？
   │   └─ 包含"分析""需求澄清"等关键词
   │       ├─ 全局搜索匹配需求名称
   │       ├─ POST /trigger/new-requirement/:id
   │       └─ 回复 "已启动分析"
   │
   └─ 普通聊天 → chatReply() → DeepSeek API → 回复
   │
   ▼
④ sendIM()
   飞书 IM API: POST /open-apis/im/v1/messages
   通过 tenant_access_token 发送
```

### Pipeline 3：OAuth 授权

```
① 同事给智小协发"授权 [文档名]"
   或自动分析发现 token 过期
   │
   ▼
② sendAuthCard()
   飞书 IM API 发送交互式卡片
   卡片按钮 URL: http://localhost:3456/auth/feishu-login
   │
   ▼
③ 同事点击卡片按钮
   浏览器打开飞书授权页
   同意授权
   │
   ▼
④ Feishu OAuth 回调
   GET /auth/callback?code=xxx
   │
   ├─ POST /open-apis/authen/v1/access_token
   │   用 code 换取 user_access_token
   │
   ├─ storeInitialToken()
   │   存入 ~/.claude/channels/feishu/wiki_token.json
   │   有效期: access_token 2h / refresh_token 14d
   │
   └─ 飞书 IM 通知 "✅ 授权成功"
   │
   ▼
⑤ 后续自动分析时
   getWikiAccessToken() → 从文件读取
   → 2h 内有效直接使用
   → 2h 过期需重新授权
```

## 三、平台对接矩阵

| 平台 | API 域名 | 鉴权方式 | 调用场景 |
|------|----------|----------|----------|
| **飞书项目** | project.feishu.cn | plugin_token (type:1) + user_key | 查询需求、搜索工作项 |
| **飞书 IM 发送** | open.feishu.cn | tenant_access_token | 发消息、发卡片 |
| **飞书 IM 接收** | open.feishu.cn | WebSocket 长连接 (SDK) | 接收聊天消息 |
| **飞书 Wiki** | open.feishu.cn | user_access_token (OAuth) | 读取知识库文档 |
| **飞书文档** | open.feishu.cn | tenant_access_token | 创建/填充飞书文档 |
| **飞书联系人** | open.feishu.cn | tenant_access_token | user_key→open_id 转换 |
| **DeepSeek** | api.deepseek.com | API Key | PRD 分析、聊天回复、Agent 解析 |

## 四、数据流向

```
飞书项目 (需求数据)
    ↓
智小协服务 (解析 + 编排)
    ├──→ 飞书 Wiki (读取 PRD)
    ├──→ DeepSeek (AI 分析)
    ├──→ 飞书文档 (创建报告)
    └──→ 飞书 IM (通知相关人员)

配置 & 状态存储:
├── config.env           ← 平台凭据
├── config/admins.json   ← 管理员
├── agents/*.json       ← Agent 配置
├── wiki_token.json     ← OAuth Token
└── prd_check_state.json ← 轮询状态
```

## 五、错误处理

| 环节 | 异常 | 处理 |
|------|------|------|
| PRD 无权限 | 131006 / Token 过期 | 自动发授权卡片，等待授权 |
| DeepSeek 超时 | 120s timeout | 返回降级提示 |
| Agent 执行崩溃 | try-catch | 独立隔离，记录日志，不影响其他 Agent |
| 飞书文档创建失败 | API 错误 | 重试 1 次 → 记录日志 |
| IM 发送失败 | 网络错误 | 静默失败，不影响主流程 |
| 飞书项目不可用 | 500 | 记录日志，下次轮询重试 |
