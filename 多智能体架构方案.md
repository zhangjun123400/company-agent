## 二、如何新增一个 Agent

### 方式一：飞书对话（管理员专用）

给智小协发一条消息：

```
新增智能体：名称=硬件方案评审，节点=硬件方案设计，功能=读取硬件方案文档，从EMC、散热、结构三个方面进行评审，输出评审意见到飞书文档，部门=硬件部
```

**全自动链路**：

```
管理员在飞书发消息
  → 智小协收到（WebSocket）
    → permission.ts 校验：sender 在 admins.json 中？
      ├─ 否 → 回复"仅管理员可新增智能体"
      └─ 是 ↓
    → DeepSeek 解析消息 → 提取 4 字段 + 补全其余配置
      → registry.ts 调用 register(config)
        ├─ 写入 agents/hardware/hw-reviewer.json
        ├─ 更新 agents/_registry.json
        ├─ 重建 dispatcher 的 节点→Agent 映射表
        └─ 回复"✅ 硬件方案评审 已注册，挂载节点：硬件方案设计"
```

### 方式二：直接放 JSON 文件（开发人员）

在 `agents/hardware/` 下新建 `hw-reviewer.json`：

```json
{
  "id": "hw-reviewer-v1",
  "name": "硬件方案评审",
  "department": "硬件部",
  "node": "硬件方案设计",
  "description": "读取硬件方案文档，从EMC、散热、结构三个方面进行评审，输出评审意见到飞书文档",
  "enabled": true,
  "timeout_ms": 300000
}
```

然后在 `agents/_registry.json` 中加一行：

```json
{
  "agents": [
    "software/requirement-analyzer.json",
    "software/tech-reviewer.json",
    "hardware/hw-reviewer.json"        ← 新增这一行
  ]
}
```

**registry.ts 检测到文件变化 → 自动热加载，无需重启。**

## 三、注册全链路

### Agent 配置结构

```json
{
  "id": "hw-reviewer-v1",            // 唯一ID
  "name": "硬件方案评审",             // 显示名
  "department": "硬件部",             // 归属部门
  "node": "硬件方案设计",             // 挂载的飞书项目节点
  "description": "...",              // 功能描述（自然语言）
  "skills": ["EMC分析", "散热评审"], // DeepSeek 自动提取
  "tools": ["wiki:read", "docx:create"],
  "output": "feishu_doc",            // 输出格式
  "output_target": ["节点负责人"],     // 发给谁
  "enabled": true,                   // 是否启用
  "timeout_ms": 300000,              // 超时
  "created_by": "ou_xxx",
  "created_at": "2026-07-04T10:00:00Z"
}
```

### 运行时流程

```
飞书项目节点流转
  → Webhook/轮询 → src/index.ts 收到事件
    → orchestrator.ts.dispatch(nodeName, workItemId)
      → dispatcher.getAgentsForNode("硬件方案设计")
        → 返回 [hw-reviewer]
          → sandbox.ts.execute(agent, context)
            ├─ 读取 PRD / 方案文档
            ├─ 调用 DeepSeek 分析
            ├─ 创建飞书文档
            └─ 发送消息通知
```

### 隔离保证

```
sandbox.ts 对每个 Agent：

1. 上下文深拷贝 → deepClone(context)
2. 独立 try-catch → 不抛给上层
3. 5分钟超时   → Promise.race([exec(), timeout])
4. 独立输出    → 各自飞书文档
```

## 四、文件依赖关系

```
src/index.ts
  ├── 启动时调用 → registry.ts.loadAll()       ← 加载所有 Agent 配置
  │                  └── 遍历 agents/_registry.json
  │                      └── 逐个 require 子目录中 JSON
  ├── 启动时调用 → dispatcher.ts.rebuild()       ← 构建 node→agent 映射
  │                  └── 依赖 registry.getAll()
  ├── 运行时      → orchestrator.ts.dispatch()
  │                  ├── 依赖 dispatcher
  │                  └── 依赖 sandbox
  └── 管理API     → permission.ts.check(openId)
                     └── 依赖 config/admins.json
```
