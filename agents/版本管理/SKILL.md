# 版本管理智能体

不调大模型，基于飞书项目 API 数据生成结构化报告。

## 触发方式
- 自动：版本定版时（人力盘点+排期通知）+ 每天 10:30/16:00（进度偏离）
- 手动：飞书发「版本盘点」

## 输出规则
所有报告：生成 MD → 上传飞书云空间 → 授权 → 发卡片通知（当前接收人：张军）。

---

## 报告一：人力盘点报告

**触发**：版本「版本需求整理」节点完成后，一次性

**数据来源**：版本开发 + 研发任务管理 + 风险管理 API

**章节**：
1. **版本全景** — 所有版本列表，各处于什么阶段，叶子任务完成数/总数
2. **人力分布（按模块）** — 各模块人员、任务数、负载标记（高负载≥3任务）
3. **人员复用率** — 每个人跨几个版本，百分比，串行风险标记
4. **版本节奏** — 已完成节点耗时（天数），当前活跃节点进度
5. **NUDD 风险管理** — 关联风险列表（等级/分值/NUDD/模块/状态）
6. **SRD 层级树** — 递归展开所有 SRD，标注模块和完成状态
7. **综合风险提示** — 门禁阻塞、高负载、未分配任务等

**SRD 完成判定**：检查叶子节点的「已完成」workflow node 是否为 status=3，**不使用**「是否完成」字段。

---

## 报告二：版本排期通知

**触发**：版本「版本需求整理」节点完成后，一次性

**数据来源**：版本开发 API（workflow_nodes）

**章节**：
1. **版本排期** — 所有节点列表（✅已完成/🔄进行中/⏳待开始），含时间
2. **SRD 叶子任务清单** — 每个叶子任务的名称、模块、负责人、完成状态

**节点时间**：已完成用 actual_finish_time，进行中用 actual_begin_time，待开始估算。

---

## 报告三：进度偏离报告

**触发**：每天 10:30 / 16:00 定时检查，偏离 >15% 时生成

**数据来源**：版本开发 API + 研发任务管理 API

**章节**：
1. **进度对比** — 当前节点、计划天数 vs 已过天数、预期进度 vs 实际进度、偏离值
2. **未完成叶子任务** — 每个未完成叶子任务的名称、模块、负责人、当前节点

**计算公式**：
```
预期进度 = (已过天数 / 计划天数) × 100%（上限 100%）
实际进度 = 已完成叶子数 / 总叶子数 × 100%（递归叶子节点判定）
偏离 = 实际进度 − 预期进度
偏离 ≤ −15% → 触发
```

**节点时长配置**：在 `config/reminder-rules.json` 的 `nodeDurations` 中定义。
